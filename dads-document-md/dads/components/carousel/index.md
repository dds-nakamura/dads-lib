## 概要

カルーセルは、複数のスライドを順番に見ることのできるコンポーネントです。ウェブサイトのアイキャッチとして使用されますが、使用には注意が必要です。

## ユースケース

### 1. シンボリックグラフィックの展開

ファーストビューの最上部に打ち出しタイプとして配置可能です。`<title>` 要素がカルーセルの見出しとして機能する場合に限定されます。ウェブサイトを象徴するグラフィックやシンプルなメッセージをバナー形式で訴求できます。

### 2. トピックコンテナとしての使用

「新着情報」や「開催中のイベント」といった特定主題のコンテンツ群に適切な見出し（`<hn>` 要素）を付けてコンテナタイプで展開します。ギャラリーコンテンツとしても利用可能です。

## 注意が必要なケース

### きちんと読んで欲しい情報を格納しない

ブラウザ上に表示されることと、ユーザーが内容を読むことは等価ではありません。統計的に2つ目以降のスライドはほぼ未閲覧です。カードなど内容が隠れない表現を推奨します。

### スライドを自動再生させない

本デザインシステムは自動再生機能を備えていません。ユーザーの自律的選択による切り替えが重要です。

### 複数の異なるトピックを同居させない

予測困難なコンテンツはユーザー滞在時間を減少させ、ブランドイメージ阻害の可能性があります。

### ユースケース以外での使用を避ける

カードや見出し・リスト要素でのレイアウト設計を検討してください。

## 仕様

### カルーセルの種別

#### 打ち出しタイプ（Key Visual Type）

「象徴的なグラフィックやプライマリメッセージ」をページ上部に表示します。`<title>` 要素がカルーセルの見出しとして機能します。マルチとシングルのバリエーションがあります。

#### コンテナタイプ（Container Type）

「新着情報やイベントなどの特定トピックのコンテナ」として使用します。カルーセルの前に見出し（`<hn>` 要素）が必要です。マルチバリエーションのみ利用可能です。

### ページレイアウト

- カルーセルの幅は通常「デフォルトコンテンツエリアの幅」を埋める
- デフォルトコンテンツエリアより小さい場合は幅を狭くすることも可能

### スライド画像の要件

- テキストは320pxまで縮小されても大きく読みやすいサイズにする
- 情報は内側2pxのボーダーエリアを避けて配置する
- テキストと背景のコントラスト比: 最低 4.5:1
- アイコン・グラフィックと背景のコントラスト比: 最低 3:1
- 写真上のテキストには背景帯またはアウトラインを使用してコントラストを確保する

### マルチカルーセル

**スライド数:** 2〜6枚

**レスポンシブ対応:** コンテナ幅に応じてスタンダードサイズと幅狭サイズを切り替え

#### スタンダードサイズ

**構成要素:**

- 見出し（コンテナタイプのみ）
- メインエリア（スライドを表示）
- ネクストエリア（次スライドのプレビューをぼかし効果付きで表示）
- ネクストボタン（スライドを進める）
- 「すべてのスライド」ボタン（スライド一覧を展開）
- ステップナビゲーション（数値インジケーター）

**レイアウト寸法（1024px例）:**

| エリア | サイズ |
|--------|--------|
| 左右マージン | 各48px |
| メインエリア | 696px |
| ネクストエリア | 232px |
| メイン:ネクスト比 | 3:1 |

**動作:**

- ステップナビゲーションは現在位置を表示。矢印キーで切り替え可能
- スライドは先頭/末尾でループする
- 「すべてのスライド」ボタンで縦型ステップナビゲーションに展開し、スライド全覧を下部に表示

#### 幅狭サイズ

**発動条件:** コンテナ幅がカルーセル幅を下回った場合

**構成要素:**

- 見出し（コンテナタイプのみ）
- メインエリア
- ページナビゲーション（前/次矢印、「現在/合計」表示）
- 「すべてのスライド」ボタン

**動作:**

- ネクストエリアのプレビューなし
- 幅狭サイズではネクストエリアがないため、必要に応じて画像ぼかしで隙間を埋める
- 最小幅サポート: 320px
- 画像はアスペクト比を維持してスケーリング

### シングルカルーセル

「打ち出しタイプのみのシングルメッセージキービジュアル」に使用します。

**構成要素:**

- メインエリアのみ（ナビゲーションUIなし）

**動作:**

- スライドにリンクを設定するかどうかを選択可能。リンク先を設定するとスライド全体がリンクになる
- スライド切り替えUI要素は表示されない
- 画像はコンテンツエリアの幅を埋める。エリアが画像より広い場合はぼかしで隙間を埋める
- エリアが画像より狭い場合は画像を比例スケーリング

## アクセシビリティ

### WCAG 2.2 / JIS 2016 準拠事項

| 達成基準 | 内容 |
|---------|------|
| 1.1.1 非テキストコンテンツ | 画像の代替テキスト |
| 2.1.1 キーボード | キーボードによるフルアクセシビリティ |
| 2.2.1 タイミング調整可能 | コンポーネントは時間制限を排除 |
| 2.2.2 一時停止・停止・非表示 | 自動再生機能を実装しない |
| 2.3.3 インタラクションによるアニメーション | アニメーションは設計上無効化 |
| 2.5.7 ドラッグ操作 | ドラッグ機能を省略 |
| 4.1.3 ステータスメッセージ | `aria-live` でスライド切り替えを通知 |

### 実装上のポイント

- `aria-live` リージョンを使用してスライド遷移をスクリーンリーダーに通知
- ステップナビゲーションには `tablist` および `tab` ロールを適用
- マルチスライドナビゲーションには `aria-label` と視覚的に非表示のラベルで識別

### カルーセルを使用するべきかを慎重に検討する

カルーセルには本質的なアクセシビリティ上の課題があります。一度に表示されるスライドがひとつであり、それ以外のスライドの内容は隠された状態となります。本コンポーネントはこれらの課題に対応する実装を提供しています。

### スライド内のテキストを短くする

読みやすさの問題を防ぎ、スクリーンリーダーの負担を軽減するためにスライドのテキストは簡潔に保ちます。

### スライドの数を少なくする

最大6枚に限定します。後半のスライドはユーザーエンゲージメントが著しく低下します。
